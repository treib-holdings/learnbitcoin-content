## 1. 三种网络形态

在 1960 年代初，RAND 的一位工程师 Paul Baran 试图设计一种能在核打击中存活的通信网络。他画了三种形态：

- **中心化。** 一个枢纽。每个节点都连到枢纽。失去枢纽，失去网络。最高效的设计，也最脆弱。
- **分散式。** 几个枢纽，各有自己的节点簇。失去一个枢纽，一个簇熄灭，其余存活。仍然有枢纽，只是更多了。
- **分布式。** 没有枢纽。每个节点直接连接几个邻居。网络可以失去大片自身，幸存者继续通信。

我们随意称之为"去中心化"的大多数东西实际上是中间选项 - 一个联邦。真正的分布比这个词暗示的要少得多。比特币声称的是第三种形态：分布式，在强意义上。

## 2. "去中心化"不意味着什么

这个词被滥用了。一些被称为"去中心化"但实际并非如此的例子：

- **"它在区块链上，所以是去中心化的。"** 大多数区块链只有少量验证者，有时真的只有几个。把数据库放到区块链上不会使数据库去中心化。只是让它更慢。
- **"多家公司运营。"** 联邦不是分布。Visa、Mastercard 和 SWIFT 都是"联邦制"的，有很多成员机构，但如果其中几个协调行动，它们就控制了网络。这不是比特币提供的。
- **"开源。"** 开放代码是去中心化的必要条件但不产生去中心化。很多开源项目由一家公司或一个基金会控制。
- **"由代币持有者治理。"** 按持有量加权的投票只是加了几个步骤的财阀统治。少数大持有者实际上决定结果。

去中心化*实际上*意味着什么，在比特币的意义上：**没有单一方面 - 或协调的小团体 - 能强制改变规则、审查用户或关闭网络。** 测试不是谁在股权表上。是谁可以被胁迫。

## 3. 比特币在什么位置

比特币是人类有史以来最去中心化的货币系统。三类参与者，各扮演不同角色，互相制衡。

**节点操作者。** 任何时刻大约有 17,000 到 22,000 个可达的比特币节点在线，算上防火墙后不可达的节点估计有 50,000 到 100,000+ 个。任何人运行验证每笔交易和区块的比特币软件。每个节点执行的规则定义了在那个节点的链副本上比特币*是什么*。如果你不喜欢一个提议的改变，你不运行新软件，你的节点继续按旧规则运行。一个节点可以在笔记本电脑、树莓派或任何有磁盘空间的廉价电脑上运行。无需许可。实时计数：[bitnodes.io](https://bitnodes.io)。

**矿工。** 竞争添加区块以获取奖励的专用硬件。地理分布在 美国、俄罗斯、哈萨克斯坦、加拿大、西欧和其他地方。他们产出区块，但不制定规则。如果矿工产出一个违反共识的区块，每个节点都会拒绝它，矿工的电力就浪费了。**矿工跟随节点，而不是反过来。**

**开发者。** 编写软件的人。[Bitcoin Core](/glossary/bitcoin-core) 是主导实现，但 Bitcoin Knots、btcd 和 Libbitcoin 是同一协议的独立实现。开发者可以提议改变；他们不能强加改变。一个提交到 Bitcoin Core 的 pull request 需要说服数以千计的节点操作者去实际运行新代码。十六年来，Bitcoin Core 有超过一千名贡献者，没有任何单一贡献者的提交超过代码的百分之几。

**集中度隐忧，以及结构性回应。**

矿池集中度：前几大矿池控制了全球算力的大部分。这听起来令人担忧，直到你意识到**矿池不拥有算力**。个别矿工（每个矿池通常有数千人）把硬件指向给他们最好条件的矿池。如果一个矿池试图攻击网络，矿工会在几天内离开。2014 年 [GHash.io](https://en.bitcoin.it/wiki/GHash.IO) 接近 51% 时我们正是看到了这一点，矿工在几周内自愿迁移出去。

实现集中度：Bitcoin Core 按节点数占主导。如果 Core 开发者被俘获，这是真实风险。缓解措施是替代实现的存在（需要时可以迅速推广）以及 Core 的审查过程涉及数十名有拒绝合并权力的独立贡献者。

这些不是完美的保护。它们比人类尝试过的任何其他货币系统都要好。

## 4. 2017 年的接管企图

比特币去中心化最清晰的压力测试发生在 2017 年。

一个由大型矿池、交易所和比特币公司组成的联盟签署了**纽约协议**（New York Agreement，NYA）。计划：激活 [SegWit](/glossary/segwit-segregated-witness-bip-141) *外加*硬分叉将区块大小翻倍，打包成称为 SegWit2x 或简称"2x"的方案。签署方代表了大约 80% 的挖矿算力和行业中的大多数主要企业。

这看起来像压倒性的共识。它不是。

一个叫做 **UASF**（用户激活软分叉，围绕 [BIP 148](https://github.com/bitcoin/bips/blob/master/bip-0148.mediawiki) 组织的草根运动）并行运行。数千名节点操作者承诺在 2017 年 8 月 1 日之前强制执行 SegWit，*不管*矿工是否同意，并拒绝不支持 SegWit 的矿工产出的区块。UASF 支持者戴着黄色帽子，更换 Twitter 头像。营销是有点傻。协调的网络立场不是。

实际发生了什么：

- SegWit 于 2017 年 8 月 23 日顺利激活。面对 UASF 威胁的潜在链分裂，挖矿多数让步了。
- 2x 硬分叉定于 2017 年 11 月，在计划激活前三天被**取消**。一旦节点操作者明确表示他们不会运行新软件，协议就崩溃了。签署方有算力；有钱；他们的节点仍然运行旧规则。他们无法实际强制改变。
- 2017 年早些时候的 8 月 1 日，另一组分叉出去创建了比特币现金（Bitcoin Cash），区块更大。它现在是一个独立网络，市值只是比特币的一小部分。

**教训在机制上。** 矿工有算力。企业有钱。开发者有注意力。这些群体中没有一个能决定。决定者是数千名选择运行什么软件的节点操作者。在笔记本电脑上运行节点的同一个人，就是其集体选择决定比特币是什么的同一批人。

这就是"没有人在负责"在实践中的样子。不是没有规则。规则极其严格。只是没有可以改变规则的中央方。

## 5. 如何衡量

去中心化很难量化。几个不完美的指标，放在一起，构成了一幅图景：

| 指标 | 比特币 | 它告诉你什么 |
|---|---|---|
| 可达节点 | 17,000-22,000 | 执行规则的参与者下限 |
| 估计总节点 | 50,000-100,000+ | 包括不可达的真实数量 |
| 挖矿算力，第一大矿池 | 变化，约 25-35% | 最大的单一出块协调者 |
| 挖矿算力，前五大矿池 | 变化，约 80-90% | 集中度脆弱性（被矿工流动性缓解） |
| 挖矿地理分布 | 8+ 个国家有重要份额 | 司法管辖区分布 |
| 独立客户端实现 | 4+（Core、Knots、btcd、Libbitcoin） | 代码层的多样性 |
| Bitcoin Core 贡献者 | 16 年来 1,000+ | 开发中没有单点故障 |

相比之下，大多数"去中心化"链有 10 到 200 个验证者，运行单一客户端，有一个有效控制路线图的基金会或团队。

这些指标不是决定性的。放在一起，它们描绘了一幅图景：比特币是有史以来最分布式的货币系统，遥遥领先，在特定层面有真实的集中度风险，社区积极努力管理。

## 6. 去中心化表演

很多项目声称"去中心化"，同时运行在介于数据库和寡头之间的东西上。这个批评是公平的，一致地应用也很重要，包括对比特币。

一些常见的去中心化表演模式：

- **"无需许可。"** 任何人都可以加入，但一小群人写了所有升级。
- **"由代币持有者治理。"** 一小撮大持有者实际上控制了治理投票。
- **"验证者集合是开放的。"** 开放的前提是你质押 $X 百万并在特定数据中心运行特定硬件。
- **"代码在 GitHub 上。"** 是的，但 pull request 只由一个基金会的员工审查和合并。
- **"这是比特币的 L2。"** 有时是真的，有时是一个自称比特币关联的联邦。

这些都可以是有用系统的特性。它们不是比特币设计所追求的，它们通不过"谁可以被胁迫"测试。

比特币的主张是可测试的：任何设备上的任何节点操作者都可以自己验证链，运行自己的软件，拒绝不接受的改变。标准是"无需许可、无需协调，只有数学和你选择运行的软件。"

这就是标准。对于稳健货币这是正确的标准。更弱的不是替代品。只是不同的产品。

## 7. 为什么难以取代

去中心化是昂贵的。它牺牲了吞吐量、能源、开发便利性和用户体验。大多数区块链在它上面妥协以获取速度和功能。

对许多用例来说这是合理的权衡。对*稳健货币*来说这是致命的权衡，因为稳健货币的全部工作是成为一个没有人可以贬值、冻结、没收或单方面升级的货币存储。一旦某一方面可以改变规则，货币就不再稳健。

如果你接受货币独特地值得防御俘获，你就接受去中心化的成本来防御它。比特币就是这个选择的实现。

代价是真实的，代价就是产品。

> **实用提示：** 当有人告诉你一个项目是去中心化的，问："如果十个特定的人协调行动，他们能改变规则、冻结资金或关闭它吗？"对大多数被称为区块链的东西，诚实答案是能。对比特币，经过十六年的压力测试后，答案仍然是不能。这就是整个产品。
